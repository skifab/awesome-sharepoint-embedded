# Contributing to Awesome SharePoint Embedded

Thank you for your interest in contributing to Awesome SharePoint Embedded! This guide will help you understand how to contribute effectively.

## How to Contribute

1. **Fork the repository** - Create your own fork of the repository.
2. **Make your changes** - Add your awesome SharePoint Embedded resource.
3. **Submit a pull request** - Open a PR with a clear description of your addition.

## Adding to the List

Before adding a new item to the list, please ensure it meets the following criteria:

### Quality Standards

- **Awesome**: The resource should be genuinely awesome and valuable to the SharePoint Embedded community.
- **Active**: Applications should be actively maintained. Links should be working.
- **Relevant**: The resource should be directly related to SharePoint Embedded.
- **Unique**: Check that the resource isn't already listed.

### Format Guidelines

#### For Applications

```markdown
- **[Application Name](https://homepage.com/)** - Brief description of what the application does.
  - [Homepage](https://homepage.com/)
  - [Microsoft AppSource](https://appsource.microsoft.com/...)
```

#### For Learning Resources

```markdown
- **[Resource Title](https://url.com/)** - Brief description of the resource content.
```

### Categories

Please add your contribution to the appropriate category:

- **Microsoft First-Party Applications**: Official Microsoft applications built on SharePoint Embedded
- **ISV Applications**: Third-party commercial applications
- **Learning Resources**: Documentation, tutorials, blog posts, videos
- **Developer Tools**: SDKs, extensions, samples, testing tools
- **Community**: Forums, discussion groups, community channels

### Pull Request Guidelines

1. **One resource per pull request** - This makes it easier to review and discuss individual additions.

2. **Clear PR title** - Use a descriptive title like:
   - "Add [Application Name] to ISV Applications"
   - "Add [Tutorial Title] to Learning Resources"
   - "Update [Resource Name] link"

3. **Provide context** - In your PR description, include:
   - Why this resource is awesome
   - How you discovered it
   - Any relevant background information

4. **Check your links** - Ensure all URLs are correct and working.

5. **Maintain alphabetical order** - Within each category, keep entries in alphabetical order (except for Microsoft First-Party Applications where logical grouping may apply).

6. **Follow the existing format** - Match the style and formatting of existing entries.

7. **Update the Table of Contents** - If you add a new section, update the table of contents.

### What NOT to Submit

- **Broken links** - All links must be working and accessible.
- **Self-promotion without value** - While self-promotion is allowed, the resource must provide genuine value.
- **Duplicate entries** - Check if the resource is already listed.
- **Unrelated resources** - Must be specifically about or built with SharePoint Embedded.
- **Paid resources without free alternatives** - Prefer free or freemium resources when possible.
- **Resources in preview/beta** - Wait until they're generally available (exceptions for significant Microsoft announcements).

## Updating Existing Entries

If you notice outdated information or broken links:

1. Create a PR with the fix
2. Explain what was wrong and how you fixed it
3. Provide evidence of the correct information if applicable

## Removing Entries

If a resource is no longer relevant, active, or available:

1. Open an issue first to discuss the removal
2. Provide justification (e.g., "Link has been dead for 6+ months", "Application discontinued")
3. After discussion, submit a PR with the removal

## Code of Conduct

- Be respectful and considerate in all interactions
- Provide constructive feedback
- Focus on the quality and relevance of resources
- Assume positive intent

## Questions?

If you have questions about contributing, feel free to:

- Open an issue for discussion
